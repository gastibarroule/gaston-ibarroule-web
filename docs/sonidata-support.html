How to Record

  1. 1Tap the center record button (red circle) on the bottom tab bar.
  2. 2A recording panel appears at the bottom with a live waveform, timer, and format badge.
  3. 3Tap the stop button to finish recording.
  4. 4The Naming View automatically opens so you can tag the recording immediately.

The app records in the background ‚Äî you can switch to other apps without stopping. The waveform display shows real-time audio levels so you can monitor your input.

The Naming View is the core of Sonidata. It's where you assign UCS-compliant metadata to your recordings using the industry-standard Universal Category System (UCS v8.2).

UCS Filename Format

{CatID}{UserCategory}_{VendorCategory}{FXName}_{CreatorID}_{SourceID}_{Location-UserData}

Example: ANMLDog_Bark-Close_GI_MKH8060_Park.wav

How to Tag a Recording

  1. 1Search for a category using SmartSearch ‚Äî type "dog" and it finds ANIMALS > DOG even if you don't know the exact name.
  2. 2Select from the dual-wheel picker (Category on left, Subcategory on right).
  3. 3Fill in the FX Name, or use Voice Slate to dictate it.
  4. 4Add Creator ID and Source ID ‚Äî enable "Remember IDs" in Settings to save across sessions.
  5. 5Add Location ‚Äî tap the location button to auto-fill from GPS, or type manually.
  6. 6Attach a Photo if desired (tap the camera icon).
  7. 7Preview the filename in the bottom bar, then tap "Rename File".

SmartSearch

The category search uses Apple's Natural Language framework for semantic understanding ‚Äî not just text matching. Type "rain" to find WEATHER > RAIN and related categories. Synonym matching means "puppy" finds ANIMALS > DOG. Exact matches score highest, followed by subcategory, synonyms, and semantic neighbors.

Batch Rename

Rename multiple files at once: enter Edit Mode, select files, tap "Rename". All selected files receive the same metadata and are renamed sequentially with unique indices.

Voice Slate lets you dictate the FX Name using speech-to-text instead of typing.

  1. 1In the Naming View, tap the microphone icon next to the FX Name field.
  2. 2Speak clearly ‚Äî describe the sound (e.g., "Bark Close").
  3. 3The transcription appears in the FX Name field.

Technical Details

Uses Apple's native SFSpeechRecognizer ‚Äî all processing happens on-device, completely private. Injects dynamic contextual hints from UCS categories to improve accuracy for sound design terminology. Records to a temporary PCM 16kHz mono file, then transcribes.

You can change the Voice Slate language in Settings ‚Üí Recording ‚Üí Voice Slate Language. Options include Device Default and all languages supported by Apple's Speech framework.

Voice Slate is a Pro feature.

Sonidata can automatically tag recordings with GPS location data.

  1. 1Grant location permission when prompted (or enable it in Settings ‚Üí Privacy ‚Üí Location).
  2. 2When you open the Naming View, the current location is automatically detected.
  3. 3Tap the location button to auto-fill the Location field.
  4. 4The full address, coordinates, and short address are all saved with the recording.

Sonidata offers multiple ways to browse your recordings.

All Files (List View)

A flat list of all recordings with expandable rows showing location, photo, and an inline audio player with waveform. Search by filename, sort by name/date/category, and use filter pills for category, subcategory, or location. Swipe left for Edit/Share/Delete, swipe right to Select.

Folder Browser

A grid view of your UCS folders showing folder name and file count. Tap a folder to see its contents, with the same swipe actions and expandable rows as the list view.

Map Viewer

An interactive map showing all geotagged recordings. Each pin represents a recording with GPS coordinates. Filter by category or location, and tap a pin for inline playback.

Switch between views using the segmented control at the top of the screen. Your preference is saved between sessions.

Sonidata can automatically organize your recordings into a UCS folder structure.

Configure in Settings ‚Üí Folder Organization. Enable Auto-Organize to move files to their UCS folder immediately after renaming. Use "Generate Folders" to create the complete UCS folder tree, or "Organize All" to batch-organize existing recordings.

Attach a photo to any recording ‚Äî useful for documenting the recording environment, location, or source.

  1. 1In the Naming View, tap the camera icon.
  2. 2Choose "Take Photo" or "Choose from Library".
  3. 3The photo is saved alongside the recording with a unique ID linking them.

Photos are included in ZIP exports alongside the audio files. Photo Attachments is a Pro feature.

CSV Export

Export your entire recordings database as a CSV file for import into sound library managers (Soundminer, BaseHead, Soundly, etc.). Includes all UCS metadata fields, location data, coordinates, file paths, audio format, file size, duration, sample rate, and bit depth. Access via Settings ‚Üí Export ‚Üí Export CSV.

ZIP Export

  1. 1Open the Export Filter Sheet from Settings or the file browser.
  2. 2Set filters: category, subcategory, location, creator ID, date range.
  3. 3Preview how many files match.
  4. 4Export as ZIP: includes audio files + photos + CSV metadata.
  5. 5Share via AirDrop, Files, email, or any share destination.

The Export screen shows your library stats at a glance: total recordings, categories used, and file counts.

ZIP Export is a Pro feature.

Import external audio files into Sonidata using the iOS Share Extension.

Supported Formats

.wav, .m4a, .mp3, .aiff, .aif, .caf

  1. 1In any app (Files, Voice Memos, etc.), tap Share.
  2. 2Select Sonidata from the share sheet.
  3. 3The file is copied to Sonidata's shared container.
  4. 4Next time you open Sonidata, the file is automatically moved to your Documents folder.
  5. 5It appears in your recordings list, ready for tagging.

Recordings don't appear in the list

Check that the recording is in the app's Documents folder. Pull down to refresh the list. If the file was moved manually via Files app, re-open Sonidata to trigger a rescan.

Voice Slate isn't working

Check that Speech Recognition permission is granted in iPhone Settings ‚Üí Sonidata ‚Üí Speech Recognition. Ensure you have a stable internet connection for the first download of the speech model. Try changing the Voice Slate Language in Settings if accuracy is poor.

Location isn't auto-filling

Check permission: iPhone Settings ‚Üí Sonidata ‚Üí Location ‚Üí "Always" (recommended) or "While Using". GPS accuracy is lower indoors ‚Äî step outside for a more precise reading.

Files aren't organizing into folders

Check that Folder Organization is enabled in Settings ‚Üí Folder Organization ‚Üí Mode. Ensure Auto-Organize is turned on. If files were recorded before enabling organization, use "Organize All" to batch-process them.

Export ZIP is empty

Check that your filters actually match some recordings (the preview count shows how many files will be included). Make sure the recordings have audio files on disk.

Frequently Asked Questions

Everything you need to know about setting up and using Sonidata for your field recordings.

Sonidata supports professional-grade audio recording in WAV format at 48kHz/24bit (Pro) and 44.1kHz/16bit (High). For longer sessions or when storage space is a priority, you can also record in compressed M4A format.

Sonidata fully integrates the Universal Category System (UCS v8.2) for standardizing sound effects metadata. When you tag a recording, you can assign it a Category, Subcategory, CatID, FX Name, Creator ID, Source ID, and Location. Sonidata then automatically renames the file to a UCS-compliant format (e.g., ANMLDog_Bark-Close_Creator_Mic_Location.wav) and can seamlessly organize your files into a matching folder hierarchy.

You can easily export your data by navigating to the Settings or directly from the file browser. Sonidata allows you to export filtered subsets of your library as complete ZIP archives, which include your audio files, any attached camera photos, and a comprehensive CSV file containing all UCS metadata.
